syntaxes : 
------------------------------------------------------------
	-	[n]>word
	-	[n]<word
	-	[n]>>word
------------------------------------------------------------

if n is not a file desciptor, then it is considered as an
argument to the command.

EX:
-	echo ss1 >2>file.txt
	->	syntax error near unexpected token `2'
-	echo ss1 > 2>file.txt
	->	syntax error near unexpected token `2'
echo ss1 >2 >file.txt
ss1 to file.txt
2 file is created

------------------------------------------------------------
	-	>word 2>&1
	-	[n]>[|]word
	-	[n]<<< word
	-	[n]<>word
	-	&>word
	-	>&word
	-	&>>word
	-	>>word 2>&1
	-	[n]<&word
	-	[n]>&word
	-	[n]<&digit-
	-	[n]>&digit-
------------------------------------------------------------

------------------------------------------------------------
	-	[n]<<[-]word
			here-document
		delimiter
------------------------------------------------------------

N.B
------------------------------------------------------------
If the 'word' folowwing the redirection operator expands to 
more than one word, Bash reports an error.
